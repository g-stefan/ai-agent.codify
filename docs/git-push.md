# Git Push Utility (`git-push.py`)

## Description
The `git-push.py` script acts as the finalization and upload stage for the automated agent's workspace. Its primary job is to package all local changes made by the AI agent into a single Git commit and upload them to the remote fork.

Specifically, it executes the following sequence:
1.  **Read Work Context**: It opens the compiled `work.json` file (typically generated by `agent-work-to-json.py`) to extract the "title" of the work performed.
2.  **Navigate & Fetch**: It switches to the local repository folder (defined by `WORK_PATH` and `GITEA_MAIN_REPO_NAME`) and performs a standard `git fetch --prune --prune-tags` to ensure the local remote objects map is clean.
3.  **Stage All**: It stages all new, modified, or deleted files using `git add --all`.
4.  **Commit**: It commits the staged changes using the `"title"` extracted from `work.json` as the commit message. If no title is found, it falls back to `"Work done"`.
5.  **Push**: It executes a standard `git push` to upload the newly created commit to the `origin` remote repository, finalizing the development loop.

## Configuration Options
This script configures itself entirely via **Environment Variables**. It does not accept standard command-line flags.

| Environment Variable       | Default Value              | Description |
|----------------------------|----------------------------|-------------|
| `WORK_PATH`                | `.` (Current Directory)    | The base directory where the repository folder is located. |
| `GITEA_MAIN_REPO_NAME`     | `"unknown"`                | The specific name of the repository folder inside the `WORK_PATH`. |
| `WORK_REPORT_FILENAME`     | `"work.json"`              | The file path to the JSON document containing the work summary, specifically its `"title"` field used for the commit message. |

## Exit Codes
*   **`0` (Success)**: All local changes were successfully staged, committed, and pushed to the remote repository.
*   **`Non-Zero` (Error)**: The script will halt and pass through the exit code generated by the underlying Git operations (`fetch`, `add`, `commit`, or `push`).

## Examples

### 1. Basic Execution (Defaults)
If you are running the script in an environment where the variables are already exported, and `work.json` exists in the current directory:
```sh
python git-push.py
```

### 2. Custom Environment Variables (Windows PowerShell)
To commit changes inside the `agent-tools` repository using a specific work report file:
```powershell
$env:WORK_PATH="C:\temp_build"
$env:GITEA_MAIN_REPO_NAME="agent-tools"
$env:WORK_REPORT_FILENAME="work.write\payload_for_pr.json"

python git-push.py
```

### 3. Custom Environment Variables (Windows CMD)
```cmd
set WORK_PATH=C:\temp_build
set GITEA_MAIN_REPO_NAME=agent-tools
set WORK_REPORT_FILENAME=work.write\payload_for_pr.json

python git-push.py
```

### 4. Custom Environment Variables (Linux / macOS / Git Bash)
You can define the variables inline immediately before running the script:
```sh
WORK_PATH="/tmp/workspace" \
GITEA_MAIN_REPO_NAME="api-service" \
WORK_REPORT_FILENAME="work.write/payload_for_pr.json" \
python git-push.py
```
