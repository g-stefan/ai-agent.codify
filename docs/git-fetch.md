# Git Fetch & Sync Utility (`git-fetch.py`)

## Description
The `git-fetch.py` script acts as a repository synchronization tool for an automated agent. Its primary purpose is to update the agent's local working copy—and its remote fork (`origin`)—with the latest changes from the original `upstream` repository. 

When executed, the script navigates into the local repository directory and performs a sequence of Git operations:
1.  **Fetch & Prune**: It runs `git fetch --prune --prune-tags` to download the latest objects and refs from all remotes while cleaning up stale or deleted tracking branches and tags.
2.  **Pull Upstream**: It pulls the designated main branch from the `upstream` remote into the current local branch. It uses `--allow-unrelated-histories` to avoid merge refusals on fresh forks and `--no-commit` to stage the merge manually.
3.  **Commit**: It explicitly commits the merged changes with a standard message `"Upstream"`.
4.  **Push**: Finally, it pushes the newly updated local branch back to the agent's fork (`origin`), ensuring both the local workspace and the remote fork are completely up-to-date with the main project.

## Configuration Options
This script does not accept standard command-line flags. Instead, it is configured entirely via **Environment Variables**:

| Environment Variable       | Default Value              | Description |
|----------------------------|----------------------------|-------------|
| `WORK_PATH`                | `.` (Current Directory)    | The base directory where the repository is located. |
| `GITEA_MAIN_REPO_NAME`     | `"unknown"`                | The name of the repository folder inside the `WORK_PATH` to navigate into. |
| `GITEA_MAIN_HEAD`          | `"main"`                   | The primary branch name of the upstream repository to sync with (e.g., `main` or `master`). |

## Exit Codes
*   **`0` (Success)**: The repository was successfully fetched, merged, and pushed to the origin fork.
*   **`Non-Zero` (Error)**: The script will halt and pass through the exit code generated by the underlying `git fetch`, `pull`, `commit`, or `push` commands if any of them fail.

## Examples

### 1. Basic Execution (Defaults)
If you are running the script in an environment where the variables are already exported:
```sh
python git-fetch.py
```

### 2. Custom Environment Variables (Windows PowerShell)
To sync a repository named `agent-tools` located in `C:\temp_build` using the `master` branch:
```powershell
$env:WORK_PATH="C:\temp_build"
$env:GITEA_MAIN_REPO_NAME="agent-tools"
$env:GITEA_MAIN_HEAD="master"

python git-fetch.py
```

### 3. Custom Environment Variables (Windows CMD)
```cmd
set WORK_PATH=C:\temp_build
set GITEA_MAIN_REPO_NAME=agent-tools
set GITEA_MAIN_HEAD=master

python git-fetch.py
```

### 4. Custom Environment Variables (Linux / macOS / Git Bash)
You can define the variables inline immediately before running the script:
```sh
WORK_PATH="/tmp/workspace" \
GITEA_MAIN_REPO_NAME="api-service" \
GITEA_MAIN_HEAD="main" \
python git-fetch.py
```
