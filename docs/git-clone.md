# Git Clone Utility (`git-clone.py`)

## Description
The `git-clone.py` script automates the initialization of a local Git workspace for an automated agent. It connects to a Gitea instance to perform a shallow clone (depth of 1) of the agent's forked repository. Once the clone is successful, it navigates into the newly created directory and automatically configures the original (main) repository as an `upstream` remote.

By combining the clone and upstream configuration steps, this script perfectly prepares the local environment for the agent to fetch the latest upstream changes, create feature branches, commit work, and eventually open a Pull Request against the main repository.

## Configuration Options
This script does not use command-line arguments. Instead, it relies entirely on **Environment Variables** for its configuration:

| Environment Variable       | Default Value              | Description |
|----------------------------|----------------------------|-------------|
| `WORK_PATH`                | `.` (Current Directory)    | The local directory where the repository should be cloned into. |
| `GITEA_INSTANCE`           | `"https://127.0.0.1"`      | The base URL of the Gitea instance hosting the repositories. |
| `GITEA_MAIN_REPO_OWNER`    | `"unknown"`                | The username or organization that owns the original/upstream repository. |
| `GITEA_WORK_REPO_OWNER`    | `"unknown"`                | The username of the agent/worker that owns the forked repository. |
| `GITEA_MAIN_REPO_NAME`     | `"unknown"`                | The name of the repository. |
| `GITEA_MAIN_HEAD`          | `"main"`                   | The primary branch name of the upstream repository to track (e.g., `main` or `master`). |

## Exit Codes
*   **`0` (Success)**: The repository was successfully cloned and the upstream remote was configured.
*   **`Non-Zero` (Error)**: The script passes through the exit code generated by the underlying `git clone` or `git remote add` commands if they fail.

## Examples

### 1. Basic Execution (Defaults)
If you have set up your environment variables elsewhere, you can run the script simply by calling it:
```sh
python git-clone.py
```

### 2. Custom Environment Variables (Windows PowerShell)
Setting up a workspace in a `temp_build` folder for a repository named `agent-tools`:
```powershell
$env:WORK_PATH="C:\temp_build"
$env:GITEA_INSTANCE="https://git.example.com"
$env:GITEA_MAIN_REPO_OWNER="core-team"
$env:GITEA_WORK_REPO_OWNER="ai-agent-bot"
$env:GITEA_MAIN_REPO_NAME="agent-tools"
$env:GITEA_MAIN_HEAD="main"

python git-clone.py
```
*Resulting Git setup:* It will clone `https://git.example.com/ai-agent-bot/agent-tools` into `C:\temp_build\agent-tools` and add `https://git.example.com/core-team/agent-tools` as the `upstream` remote tracking the `main` branch.

### 3. Custom Environment Variables (Linux / macOS / Git Bash)
You can define the variables inline before executing the script:
```sh
WORK_PATH="/tmp/workspace" \
GITEA_INSTANCE="https://git.mycompany.local" \
GITEA_MAIN_REPO_OWNER="backend" \
GITEA_WORK_REPO_OWNER="agent-worker-1" \
GITEA_MAIN_REPO_NAME="api-service" \
GITEA_MAIN_HEAD="master" \
python git-clone.py
```
