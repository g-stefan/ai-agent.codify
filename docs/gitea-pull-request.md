# Gitea Pull Request Utility (`gitea-pull-request.py`)

## Description
The `gitea-pull-request.py` script is a command-line tool designed to automate the creation of Pull Requests (PRs) via the Gitea REST API. It constructs the PR payload by reading title and body data from a JSON file (typically generated by `agent-work-to-json.py`) and optionally links an existing issue to the PR.

A key feature of this script is its built-in **conflict resolution logic**. If the script attempts to create a Pull Request but the API responds that a PR for those specific branches already exists, it will automatically locate the existing open PR and post the new title and body as a comment on that PR instead of failing. This makes the tool highly robust for continuous automation loops where an agent might push multiple commits to the same feature branch over time.

## Command-Line Options

The script uses `argparse` and requires explicit command-line flags.

### Required Arguments
| Argument | Description |
| :--- | :--- |
| `--url` | The base URL of the Gitea instance (e.g., `https://gitea.example.com`). |
| `--repo` | The target repository path in the format `owner/repo` (e.g., `core-team/agent-tools`). |
| `--head` | The name of the branch containing your commits (e.g., `feature-fix-bug`). |
| `--pr-json` | Path to a JSON file containing the PR content. Must have `"title"` and `"body"` keys. |

### Optional / Contextual Arguments
| Argument | Default | Description |
| :--- | :--- | :--- |
| `--fork` | `None` | The username/owner of the fork repository if making a cross-repository PR (e.g., `ai-agent-bot`). |
| `--base` | `"main"` | The target branch you want to merge into. |
| `--issue` | `None` | Explicitly provide an issue number to link (e.g., `42`). |
| `--issue-json` | `None` | Path to a JSON file containing the Gitea issue data (e.g., from `gitea-issues-get.py`). The script will parse this to automatically extract the issue number and link it. |
| `--token` | `None` | Your Gitea API access token. If omitted, the script reads the `GITEA_TOKEN` environment variable. |
| `-h`, `--help`| | Shows the help message and exits. |

*Note: `--issue` and `--issue-json` are mutually exclusive. If both are provided, `--issue` takes precedence.*

## Prerequisites
This script requires the `requests` library:
```sh
pip install requests
```

## Exit Codes
*   **`0` (Success)**: The PR was successfully created, OR the PR already existed and the content was successfully posted as a comment on the existing PR.
*   **`1` (Error)**: An error occurred, such as missing input JSON files, missing API tokens, network failures, or unhandled HTTP errors from the Gitea API.

## Examples

### 1. Basic Pull Request within the Same Repository
Create a PR merging `feature-login` into `main` using the content defined in `work.json`:
```sh
python gitea-pull-request.py \
  --url https://git.example.com \
  --repo frontend/website \
  --head feature-login \
  --pr-json work.json \
  --token my_secret_token
```

### 2. Cross-Repository (Forked) Pull Request
Create a PR from a worker's fork (`ai-agent-bot`) merging into the upstream `core-team/agent-tools` repository:
```sh
export GITEA_TOKEN="my_secret_token"
python gitea-pull-request.py \
  --url https://git.example.com \
  --repo core-team/agent-tools \
  --fork ai-agent-bot \
  --head fix-parser-bug \
  --pr-json work.write/work.json
```

### 3. Automatically Linking an Issue
Create a PR and automatically extract the issue number from the `issue.json` file to append "*Resolves #XX*" to the PR body:
```sh
python gitea-pull-request.py \
  --url https://git.example.com \
  --repo core-team/agent-tools \
  --fork ai-agent-bot \
  --head fix-issue-42 \
  --pr-json work.write/work.json \
  --issue-json work.read/issue.json
```
