# Agent Work to JSON (`agent-work-to-json.py`)

## Description
The `agent-work-to-json.py` script is a consolidation utility. It merges the raw, step-by-step activity of an expert AI agent with a finalized, high-level summary report into a single, comprehensive "work report" JSON payload. 

Specifically, the script performs the following steps:
1.  **Reads the Expert Session**: It opens the chat session JSON file and extracts all the text generated by the `"assistant"` role, concatenating it into a single, long `body` string representing the raw work done.
2.  **Reads the Summary Report**: It opens a separate JSON report file (often generated by `agent-report-to-json.py`) to extract the final `"title"` and the summary `"body"` (which is mapped to a `"comment"` field).
3.  **Generates the Payload**: It combines these elements into a cleanly formatted, indented JSON file structured with three main keys: `"title"`, `"comment"`, and `"body"`. This consolidated file is typically used to automatically create a Pull Request or update an issue tracker.

## Configuration Options
This script does not use traditional command-line flags or positional arguments. Instead, it is configured entirely through **Environment Variables**:

| Environment Variable       | Default Value     | Description |
|----------------------------|-------------------|-------------|
| `PROMPT_EXPERT_SESSION`    | `"session.json"`  | The file path to the JSON document containing the full chat session history. |
| `PROMPT_REPORT_JSON`       | `"report.json"`   | The file path to the JSON document containing the summarized report (title and body). |
| `WORK_REPORT_FILENAME`     | `"work.json"`     | The destination file path where the consolidated JSON payload will be written. |

## Exit Codes
*   **`0` (Success)**: The inputs were successfully parsed, combined, and written to the output file.
*   **`1` (Error)**: Execution halted. This occurs if the script cannot find/read the input files, if the input files contain invalid JSON, or if the script lacks permissions to write the destination file.

## Examples

### 1. Basic Execution (Defaults)
If all your files (`session.json` and `report.json`) are located in the current directory:
```sh
python agent-work-to-json.py
```
This will generate a `work.json` file in the same directory.

### 2. Custom Environment Variables (Windows PowerShell)
If your files are organized into subdirectories (e.g., `work.read` and `work.write`):
```powershell
$env:PROMPT_EXPERT_SESSION="work.write\expert_session.json"
$env:PROMPT_REPORT_JSON="work.write\final_report.json"
$env:WORK_REPORT_FILENAME="work.write\payload_for_pr.json"

python agent-work-to-json.py
```

### 3. Custom Environment Variables (Linux / macOS / Git Bash)
You can define the variables inline for a single execution:
```sh
PROMPT_EXPERT_SESSION="work.write/expert_session.json" \
PROMPT_REPORT_JSON="work.write/final_report.json" \
WORK_REPORT_FILENAME="work.write/payload_for_pr.json" \
python agent-work-to-json.py
```

### 4. Output Structure Example
Assuming your `report.json` contains a title and body, and your `session.json` contains the step-by-step chat history, the resulting `work.json` will look like this:
```json
{
    "title": "Fix memory leak in data processing module",
    "comment": "Identified the unclosed buffer stream in process_data() and added the required cleanup sequence.",
    "body": "I have executed grep_search to find process_data...\n\nI am replacing the old code with the new code..."
}
```
