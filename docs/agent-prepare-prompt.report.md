# Agent Prepare Prompt (Report) (`agent-prepare-prompt.report.py`)

## Description
The `agent-prepare-prompt.report.py` script is a text-generation utility that constructs a final Markdown prompt meant for a "Reporting" AI agent. It does this by extracting all the actions and statements made by the "Assistant" (the expert agent) from a saved JSON chat session history and sandwiching them between predefined prefix and suffix templates.

The core process of the script is as follows:
1.  **Read Begin Template**: Loads an introductory text (e.g., instructions on how to generate the report).
2.  **Extract Assistant History**: Reads a JSON file representing a previous LLM chat session (e.g., generated by `agent-llm-cli.py`). It specifically iterates over the messages and extracts only the `content` blocks where the `role` is `"assistant"`. These are appended sequentially.
3.  **Read End Template**: Loads concluding instructions (e.g., formatting constraints for the final JSON or Markdown report).
4.  **Save Output**: Writes the combined text into a final Markdown prompt file, ready to be fed to the Reporting agent.

## Configuration Options
Like other scripts in this suite, this script is configured exclusively via **Environment Variables**. It does not use traditional command-line arguments.

| Environment Variable       | Default Value     | Description |
|----------------------------|-------------------|-------------|
| `PROMPT_REPORT_BEGIN`      | `"prompt.begin.md"`| The path to the introductory portion of the reporting prompt. |
| `PROMPT_REPORT_END`        | `"prompt.end.md"`  | The path to the concluding portion of the reporting prompt. |
| `PROMPT_EXPERT_SESSION`    | `"session.json"`   | The path to the JSON file containing the prior chat session history (the source data). |
| `PROMPT_REPORT_FILENAME`   | `"prompt.md"`      | The path to the output file where the finalized prompt will be written. |

## Exit Codes
*   **`0` (Success)**: The reporting prompt was successfully created and written to disk.
*   **`1` (Error)**: The script encountered a fatal issue, such as missing input files, invalid JSON parsing in the session file, or insufficient write permissions for the output file.

## Examples

### 1. Basic Execution (Defaults)
If you rely entirely on default file names in your current directory:
```sh
python agent-prepare-prompt.report.py
```

### 2. Custom Environment Variables (Windows PowerShell)
To explicitly map input templates, the generated session data, and the final output destination:
```powershell
$env:PROMPT_REPORT_BEGIN="work.read\prompt.report.begin.md"
$env:PROMPT_REPORT_END="work.read\prompt.report.end.md"
$env:PROMPT_EXPERT_SESSION="work.write\expert_session.json"
$env:PROMPT_REPORT_FILENAME="work.write\final_report_prompt.md"

python agent-prepare-prompt.report.py
```

### 3. Custom Environment Variables (Windows CMD)
```cmd
set PROMPT_REPORT_BEGIN=work.read\prompt.report.begin.md
set PROMPT_REPORT_END=work.read\prompt.report.end.md
set PROMPT_EXPERT_SESSION=work.write\expert_session.json
set PROMPT_REPORT_FILENAME=work.write\final_report_prompt.md

python agent-prepare-prompt.report.py
```

### 4. Custom Environment Variables (Linux / macOS / Git Bash)
You can set the environment variables right before executing the script:
```sh
PROMPT_REPORT_BEGIN="work.read/prompt.report.begin.md" \
PROMPT_REPORT_END="work.read/prompt.report.end.md" \
PROMPT_EXPERT_SESSION="work.write/expert_session.json" \
PROMPT_REPORT_FILENAME="work.write/final_report_prompt.md" \
python agent-prepare-prompt.report.py
```
