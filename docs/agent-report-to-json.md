# Agent Report to JSON (`agent-report-to-json.py`)

## Description
The `agent-report-to-json.py` script is a parsing utility that converts a standard Markdown text file into a structured JSON document. It is typically used to take the final human-readable report generated by an AI agent and package it into a machine-readable JSON payload, perhaps to be sent back to an API, issue tracker, or database.

When executed, the script scans the provided Markdown file:
1. It identifies the **first non-empty line** and designates it as the **Title**. Any leading Markdown header characters (e.g., `#`) or whitespace are automatically stripped.
2. All subsequent lines are grouped together, stripped of trailing/leading blank lines, and designated as the **Body**.
3. It outputs these two pieces of data into a cleanly formatted, indented JSON file under the keys `"title"` and `"body"`.

## Command-Line Options
Unlike other environment-variable-driven scripts in this repository, this script utilizes standard positional command-line arguments.

| Argument | Type | Description |
| :--- | :--- | :--- |
| `input_file` | *Positional* | The path to the source Markdown (`.md`) file you wish to parse. |
| `output_file` | *Positional* | The path to the destination JSON (`.json`) file where the result will be saved. |
| `-h`, `--help`| *Flag* | Shows the help message and exits. |

## Exit Codes
*   **`0` (Success)**: The markdown file was successfully parsed and the JSON file was written.
*   **`1` (Error)**: Execution halted due to an I/O error (e.g., the input file was not found, or the script lacks permissions to write the output file). Errors are printed to standard error (`stderr`).

## Examples

### 1. Basic Execution
Convert a report named `report.md` into `report.json` in the current directory:
```sh
python agent-report-to-json.py report.md report.json
```

### 2. File Path Execution
Convert a file from a reading directory and output it to a writing directory:
```sh
python agent-report-to-json.py work.read/final_report.md work.write/payload.json
```

### 3. Understanding the Output Structure
If your input file (`report.md`) looks like this:
```markdown
# AI Code Review Summary

This is the first paragraph of the review.
- It found 2 bugs.
- It suggested 1 refactor.
```

Running the script will generate an `output.json` that looks like this:
```json
{
    "title": "AI Code Review Summary",
    "body": "This is the first paragraph of the review.\n- It found 2 bugs.\n- It suggested 1 refactor.\n"
}
```
